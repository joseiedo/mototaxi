    }

        rdb := redis.NewClient(&redis.Options{Addr: addr})
        return &redisStore{client: rdb}, nil
    }

    WriteLocation:
    - Start pipeline: `pipe := s.client.Pipeline()`
    - `pipe.GeoAdd(ctx, "drivers:geo", &redis.GeoLocation{Name: driverID, Longitude: lng, Latitude: lat})`
    - `pipe.Set(ctx, "driver:"+driverID+":latest", payload, 30*time.Second)`
    - `_, err := pipe.Exec(ctx)` — if err != nil return err (no partial success, per CONTEXT.md decision)
    - Do NOT use TxPipeline (MULTI/EXEC) — no atomicity benefit, extra overhead (see RESEARCH.md Pattern 3)

    ReadLocation:
    - `val, err := s.client.Get(ctx, "driver:"+driverID+":latest").Result()`
    - if `err == redis.Nil` → return nil, ErrNotFound
    - if other err → return nil, err
    - return []byte(val), nil

    Ping:
    - `s.client.Ping(ctx).Err()`

    Note: `ErrNotFound` must be an exported sentinel so the handler can distinguish "not found" from "service error".
    Cohesion rule: WriteLocation and ReadLocation belong in this package — do not scatter Redis key construction to the handler.

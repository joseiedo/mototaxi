Nginx decisions (locked in CONTEXT.md):
- Single file: nginx/nginx.conf (not conf.d/ split)

- Route /location → location_service, /socket/websocket → push_server, /nginx_status → stub_status
- proxy_read_timeout 3600s on WebSocket location
- X-Upstream-Addr response header on all proxy location blocks
- resolver 127.0.0.11 valid=30s in http block

- stub_status allow 172.16.0.0/12, deny all (Docker internal access only)
- NO metrics proxy through Nginx — Prometheus scrapes services directly

  <name>Task 1: Create nginx/nginx.conf</name>
  <files>nginx/nginx.conf</files>
  <action>
Create the directory `nginx/` at the project root and write `nginx/nginx.conf` with the following exact structure:

```nginx
worker_processes auto;

events {
    worker_connections 1024;
}

http {
    resolver 127.0.0.11 valid=30s;

    upstream location_service {
        least_conn;
        server location-service:8080;
        keepalive 32;
    }

    upstream push_server {
        ip_hash;
        server push-server:4000;
        keepalive 32;
    }

    server {
        listen 80;

        # HTTP: location-service replicas (least_conn)
        location /location {
            proxy_pass            http://location_service;
            proxy_http_version    1.1;
            proxy_set_header      Connection "";
            proxy_set_header      Host $host;
            proxy_set_header      X-Real-IP $remote_addr;
            add_header            X-Upstream-Addr $upstream_addr always;
            proxy_connect_timeout 5s;
            proxy_send_timeout    30s;
            proxy_read_timeout    30s;
        }

        # WebSocket: push-server replicas (ip_hash sticky)
        location /socket/websocket {
            proxy_pass            http://push_server;
            proxy_http_version    1.1;
            proxy_set_header      Upgrade $http_upgrade;
            proxy_set_header      Connection "upgrade";
            proxy_set_header      Host $host;
            proxy_set_header      X-Real-IP $remote_addr;
            add_header            X-Upstream-Addr $upstream_addr always;
            proxy_connect_timeout 5s;
            proxy_send_timeout    3600s;
            proxy_read_timeout    3600s;
        }

        # Nginx metrics — Docker internal network only (Phase 7 scraping)
        location /nginx_status {
            stub_status;
            allow 172.16.0.0/12;
            allow 127.0.0.1;
            deny  all;
        }
    }
}
```

Key rules to verify are present:
- `Connection ""` (empty, clearing hop-by-hop header) on the /location block — NOT "upgrade"
- `Connection "upgrade"` (literal string) on the /socket/websocket block — NOT a variable
- `proxy_read_timeout 3600s` AND `proxy_send_timeout 3600s` both present on WebSocket block
- `add_header X-Upstream-Addr $upstream_addr always` on BOTH proxy location blocks (the `always` flag makes the header appear on error responses too)
- `resolver 127.0.0.11 valid=30s` in the http block (not inside an upstream block)
- `keepalive 32` in BOTH upstream blocks

**Step 1 — Write the smoke test script then bring up the stack:**

Create `infra/smoke-test-nginx.sh` with this content (make it executable with `chmod +x`):

```bash
#!/usr/bin/env bash
set -euo pipefail

echo "=== Nginx Smoke Test ==="

echo ""
echo "--- NGINX-01: POST /location distribution across replicas ---"
declare -A seen_ips
for i in $(seq 1 6); do
  ip=$(curl -s -X POST http://localhost/location \
    -H "Content-Type: application/json" \
    -d '{"driver_id":"smoke-d1","lat":-23.5,"lng":-46.6,"bearing":90.0,"speed_kmh":30.0,"emitted_at":"2026-03-07T10:00:00Z"}' \
    -o /dev/null -D - 2>/dev/null | grep -i "x-upstream-addr" | awk '{print $2}' | tr -d '\r')
  echo "  Request $i → upstream: $ip"
  seen_ips["$ip"]=1
done
distinct_count=${#seen_ips[@]}
echo "  Distinct upstream IPs seen: $distinct_count"
if [ "$distinct_count" -lt 2 ]; then
  echo "  WARN: Only $distinct_count distinct upstream IP(s) — expected ≥2 with 2 replicas"
else
  echo "  OK: Distribution confirmed across $distinct_count replicas"
fi

echo ""
echo "--- NGINX-02: WebSocket upgrade (expect 101 Switching Protocols) ---"
ws_status=$(curl -si \
  -H "Connection: Upgrade" \
  -H "Upgrade: websocket" \
  -H "Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==" \
  -H "Sec-WebSocket-Version: 13" \
  http://localhost/socket/websocket 2>/dev/null | head -1)
echo "  WebSocket response: $ws_status"
if echo "$ws_status" | grep -q "101"; then
  echo "  OK: WebSocket upgrade succeeded"
else
  echo "  NOTE: Got '$ws_status' — if push-server is not running, 502 is expected"
fi

echo ""
echo "--- nginx_status (may return 403 from host, expected) ---"
nginx_status=$(curl -si http://localhost/nginx_status 2>/dev/null | head -1)
echo "  /nginx_status response: $nginx_status"

echo ""
echo "=== Smoke Test Complete ==="
```
